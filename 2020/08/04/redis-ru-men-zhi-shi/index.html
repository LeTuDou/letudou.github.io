Redis入门知识

Redis 是完全开源免费的，遵守BSD协议，是一个高性能的key-value数据库。

内存存储和持久化

  Redis 数据库中的所有数据都存储在内存中，由于内存读写速度远远超过硬盘的读写速度,相比较其他基于硬盘存储的数据数据库有很明显的优势。
  将数据全部存在内存中比较不稳妥，如果由于不可抗力因素主机断电数据会丢失。Redis提供了对持久化的支持，也就是将内存中的数据异步备份写入到硬盘，并提供两种方式，分别是RDB （Redis DataBase）和 AOF （Append Only File）;

Redis的安装

  1. 在Linux中的安装
    wget http://download.redis.io/releases/redis-5.0.5.tar.gz
tar xvf redis-5.0.5.tar.gz
cd redis-5.0.5
make
    make完后 redis-5.0.5目录下会出现编译后的redis服务程序redis-server,还有用于测试的客户端程序redis-cli
  2. 在Windows中的安装
    下载地址：https://github.com/dmajkic/redis/downloads 根据自己情况下载对应版本，支持32位和64位
    解压到某一目录。

文件介绍(以Windows为例)

文件名 说明
redis-server Redis服务端程序
redis-cli Redis命令行客户端
redis-benchmark Redis性能测试工具
redis-check-aof AOF文件修复工具
redis-check-dump RDB文件修复工具

Redis启动与停止

   Redis启动方式:直接运行redis-server即可启动Redis;Redis默认使用6379端口，找到配置文件第25行:port 6379可自定义修改
   Redis停止方式:

  1. 命令行结束redis-server进程
  2. redis-cli SHUTDOWN

      第一种方式会导致数据丢失,不推荐使用
      第二种方式当Redis收到 SHUTDOWN命令后,会先断开所有客户端连接,然后根据配置执行持久化,最后完成退出。Redis可以稳妥处理SIGTERM信号,所以使用killRedis进程的PID 也可以正常结束Redis,效果与发送SHUTDOWN命令一样。

Redis数据类型

  1. Redis的五大数据类型

    • String (字符串)
      • 一个key对应一个value
      • 二进制安全 redis的String可以包含任何数据，比如二进制jpg图片或者序列化的对象
      • 一个Redis中字符串value最多是512MB
    • Hash (哈希,类似Java的Map)
      • 是一个键值对集合
      • 是一个String类型的fieId 和value的映射表, Hash特别适合用于存储对象
    • List (列表)
      • 简单的字符串列表，按照插入顺序排序,添加一个元素到列表的头部(左边)或者尾部(右边)
      • 底层是一个链表
    • Set (集合)
      • 是String类型的无序集合
      • 通过HashTable实现的
    • Zset (sorted set:有序集合)
      • Zset和Set同样都是String类型元素的集合,且不允许出现重复的成员
      • 不同的是Zset每个元素都会关联一个double类型的分数
      • Zset的成员唯一,但分数却可以重复,通过分数排序

  2. Redis键(key)举例

    • EXISTS key的名字,判断某个key是否存在

      redis 127.0.0.1:6379> EXISTS k1
(integer) 1
redis 127.0.0.1:6379> EXISTS k
(integer) 0
redis 127.0.0.1:6379>

    • move key db 当前库中没有，移动到指定的库中

      redis 127.0.0.1:6379> move k3 7
(integer) 1
redis 127.0.0.1:6379> keys *
1) "k4"
2) "k1"
3) "k2"
redis 127.0.0.1:6379> SELECT 7
OK
redis 127.0.0.1:6379[7]> keys *
1) "k3"
redis 127.0.0.1:6379>

    • expire key 秒: 给指定的key设置过期时间;ttl key:以秒为单位返回 key 的剩余过期时间(在 Redis 2.8 以前，当 key 不存在，或者 key 没有设置剩余生存时间时，命令都返回 -1 )

      redis 127.0.0.1:6379[7]> SELECT 0
OK
redis 127.0.0.1:6379> keys *
1) "k4"
2) "k1"
3) "k2"
redis 127.0.0.1:6379> expire k2 10
(integer) 1
redis 127.0.0.1:6379> ttl k2
(integer) 7
redis 127.0.0.1:6379> ttl k2
(integer) 5
redis 127.0.0.1:6379> ttl k2
(integer) 4
redis 127.0.0.1:6379> ttl k2
(integer) -1
redis 127.0.0.1:6379> keys *
1) "k4"
2) "k1"
redis 127.0.0.1:6379> ttl k1
(integer) -1
redis 127.0.0.1:6379> ttl k2
(integer) -1
redis 127.0.0.1:6379>

    • type key:查看当前key的类型

      redis 127.0.0.1:6379> set k1 types
OK
redis 127.0.0.1:6379> type k1
string
redis 127.0.0.1:6379>  lpush myList 1 2 3 4 5
(integer) 5
redis 127.0.0.1:6379> type myList
list
redis 127.0.0.1:6379> LRANGE myList 0 -1
1) "5"
2) "4"
3) "3"
4) "2"
5) "1"
redis 127.0.0.1:6379>

  3. Redis字符串(String)举例

    • set/get/del/append/strlen

      redis 127.0.0.1:6379> set k1 v1
OK
redis 127.0.0.1:6379> get k1
"v1"
redis 127.0.0.1:6379> append k1 v1
(integer) 4
redis 127.0.0.1:6379> get k1
"v1v1"
redis 127.0.0.1:6379> strlen k1
(integer) 4
redis 127.0.0.1:6379> keys *
1) "k1"
redis 127.0.0.1:6379> del k1
(integer) 1
redis 127.0.0.1:6379> keys *
(empty list or set)
redis 127.0.0.1:6379>

    • incrby/decrby 一定是数字才可以进行加减

      redis 127.0.0.1:6379> set k2 0
OK
redis 127.0.0.1:6379> INCRBY k2 1
(integer) 1
redis 127.0.0.1:6379> INCRBY k2 1
(integer) 2
redis 127.0.0.1:6379> INCRBY k2 1
(integer) 3
redis 127.0.0.1:6379> decrBy k2 1
(integer) 2
redis 127.0.0.1:6379> decrBy k2 1
(integer) 1
redis 127.0.0.1:6379> get k2
"1"
redis 127.0.0.1:6379>

    • getrange/setrange

      redis 127.0.0.1:6379> set k1 aaabbbccc
OK
redis 127.0.0.1:6379> getrange k1 0 -1
"aaabbbccc"
redis 127.0.0.1:6379> getrange k1 0 3
"aaab"
redis 127.0.0.1:6379> setrange k1 3 ddd
(integer) 9
redis 127.0.0.1:6379> getrange k1 0 -1
"aaadddccc"
redis 127.0.0.1:6379>

    • setex(set with expire)键秒值/setnx(set if not exist)

      redis 127.0.0.1:6379> setex k3 10 v3
OK
redis 127.0.0.1:6379> ttl k3
(integer) 7
redis 127.0.0.1:6379> ttl k3
(integer) 6
redis 127.0.0.1:6379> ttl k3
(integer) 5
redis 127.0.0.1:6379> ttl k3
(integer) 5
redis 127.0.0.1:6379> ttl k3
(integer) 4
redis 127.0.0.1:6379> get k3
(nil)
redis 127.0.0.1:6379> get k1
"aaadddccc"
redis 127.0.0.1:6379> setnx k1 aaaaaaaa
(integer) 0
redis 127.0.0.1:6379> get k1
"aaadddccc"
redis 127.0.0.1:6379>

    • mset/mget/msetnx

      redis 127.0.0.1:6379> mset k1 v1 k2 v2 k3 v3 k4 v4
OK
redis 127.0.0.1:6379> mget k1 k2 k3 k4
1) "v1"
2) "v2"
3) "v3"
4) "v4"
redis 127.0.0.1:6379>

Redis命令行客户端使用

  1. 启动服务端
    redis-server
  2. 连接客户端
    redis-cli -h 127.0.0.1 -p 6379
  3. 连通性测试
    redis 127.0.0.1:6379> ping
PONG
redis 127.0.0.1:6379> echo hello
"hello"
redis 127.0.0.1:6379>
  4. 举例读取操作
    redis 127.0.0.1:6379> SET name "LeTuDou"
OK
redis 127.0.0.1:6379> get name
"LeTuDou"
redis 127.0.0.1:6379>
  5. 演示操作
    Redis默认16个数据库 可以通过select num切换数据库 其中num大于等于0小于等于16减1一下举例切换到第8个数据库
redis 127.0.0.1:6379> select 7
OK
redis 127.0.0.1:6379[7]> select 0
OK
redis 127.0.0.1:6379>

DBSIZE命令查看当前数据库中key的数量;keys * 查看当前数据库中key

redis 127.0.0.1:6379> DBSIZE
(integer) 1
redis 127.0.0.1:6379> keys *
1) "name"
redis 127.0.0.1:6379> set k1 v1
OK
redis 127.0.0.1:6379> set k2 v2
OK
redis 127.0.0.1:6379> set k3 v3
OK
redis 127.0.0.1:6379> keys k?
1) "k1"
2) "k2"
3) "k3"
redis 127.0.0.1:6379> DBSIZE
(integer) 4
redis 127.0.0.1:6379> keys *
1) "k1"
2) "k2"
3) "name"
4) "k3"

FLUSHDB清空当前库 FLUSHALL 清空16个库

redis 127.0.0.1:6379[7]> set v1 v17
OK

OK

OK

(empty list or set)
redis 127.0.0.1:6379>

备注:

  1. 统一密码管理:16个库都是一样的密码，要么连接16个都可以连接成功，要么一个都连不上
  2. Redis索引都是从0开始
